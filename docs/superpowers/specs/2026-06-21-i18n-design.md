# i18n — Language Selector & Full-Site Translation

**Date:** 2026-06-21  
**Status:** Approved  
**Languages:** English (default), Spanish, German, French, European Portuguese

---

## Goal

Add a language dropdown to the Dimonova marketing site so visitors can read all content in their preferred language. The selected language persists across page refreshes via `localStorage`.

---

## Architecture

Three files change, one is new. No build-step changes required.

| File | Change |
|------|--------|
| `_design_source.html` | Every translatable text node gets `data-i18n="key"`. Mixed-HTML headings (those embedding a `<em>` accent word) get `data-i18n-html="key"` on the parent element. Input placeholders get `data-i18n-placeholder="key"`. The language selector UI is added in two places (see UI section). |
| `i18n.js` (new) | `window.I18N = { en: {…}, es: {…}, de: {…}, fr: {…}, pt: {…} }`. Loaded via `<script>` tag in `index.html` **before** `app.js`. |
| `app.js` | Gains `lang` state, `langOpen` state, `setLanguage(code)`, and click-outside handling for the dropdown. `setLanguage` is called once on init and once on every language switch — not on every `render()` call. |
| `_build.js` | No changes needed. The builder already passes `data-*` attributes through untouched. |

### Initialisation sequence

1. `i18n.js` loads, sets `window.I18N`.
2. `app.js` reads `localStorage.getItem('lang')`, defaults to `'en'`.
3. `setLanguage(code)` runs immediately — walks all tagged elements and applies strings.
4. On language switch: `localStorage.setItem('lang', code)`, update `state.lang`, call `setLanguage(code)`, close dropdown.

### Translation attribute conventions

| Attribute | Used for |
|-----------|----------|
| `data-i18n="key"` | Plain text nodes (sets `textContent`) |
| `data-i18n-html="key"` | Headings containing `<em>` accent words (sets `innerHTML`) |
| `data-i18n-placeholder="key"` | `<input>` and `<textarea>` placeholder attributes |

### Mixed-HTML headings

Inline styles on `<em class="dim-h1-accent">` and similar accent elements are moved to their CSS class in `_design_source.html`'s `<style>` block. This means translation values can reference `<em class="dim-h1-accent">word</em>` without duplicating inline styles across five languages.

## Language Selector UI

### Desktop header

Placed between the desktop nav links and the WhatsApp/demo buttons. Styled as a compact pill matching the existing WhatsApp button.

```
[ [flag img] EN ▾ ]   [ ● WhatsApp ]   [ Request a demo ]
```

Flag images are served from `flagcdn.com` as 16×12 PNG files (e.g. `https://flagcdn.com/16x12/gb.png`). No emoji flags — they render inconsistently on Windows.

Clicking opens a dropdown panel (white background, `1px solid #E8E0D2` border, `border-radius: 14px`, `box-shadow` matching card components). Each row: flag image + language code + full name. Active language has a terracotta dot (`#B8523A`) on the right.

```
┌──────────────────────┐
│ [gb] EN  English   • │  ← active
│ [es] ES  Español     │
│ [de] DE  Deutsch     │
│ [fr] FR  Français    │
│ [pt] PT  Português   │
└──────────────────────┘
```

Flag CDN map: `en → gb`, `es → es`, `de → de`, `fr → fr`, `pt → pt`.

Dropdown closes on selection or click-outside (document click listener checks if target is outside the selector element).

### Mobile nav drawer

A row of five flag image + code pill buttons added at the bottom of the mobile nav drawer, above "Request a demo". Active pill uses the dark fill (`#1F1814` bg, `#FAF6F0` text), inactive pills use the ghost style (`transparent` bg, `#1F1814` text, `#E8E0D2` border) — same pattern as the venue-type pills in the contact form.

```
[ [gb] EN ]  [ [es] ES ]  [ [de] DE ]  [ [fr] FR ]  [ [pt] PT ]
```

## Translation Scope

### Translated

- All nav labels, page headings, subheadings, body paragraphs, section labels
- Button labels throughout ("Request a demo", "Chat on WhatsApp", "See all features →", etc.)
- Contact form: labels, input placeholders, validation error messages, success state text
- Footer: tagline, column headers, link labels, copyright line
- WhatsApp widget: greeting message, quick-reply chips, "Continue to WhatsApp" button
- Dashboard mock UI labels inside feature visuals ("All dishes", "Starters", "Mains", "Sold out", "Menu views", "Updates this week", "+ New dish", etc.)
- Stats labels ("Average go-live", "Time to edit a dish", "Total views", "Top section", "Trend")
- FAQ questions and answers (Pricing page)
- Placeholder editorial text (case study pull-quotes, testimonial, About paragraphs)
- Pricing page: feature lists, CTA strip copy

### Not translated

- Email address, phone number, opening hours values
- Currency amounts and numbers ("From €600", "~10 seconds", "2,184 +18%", "5–7 days")
- Brand name "Dimonova"
- Dish names in phone/dashboard mockups ("Beef & Guinness pie", etc.) — design props
- "The Crowing Cock" (example venue name in mockups)
- Terms / Privacy / Cookies labels (no real pages yet)

## `app.js` Changes

### New state fields

```js
lang: localStorage.getItem('lang') || 'en',
langOpen: false,
```

### New functions

**`setLanguage(code)`**  
Walks `[data-i18n]` → sets `textContent` from `window.I18N[code][key]`.  
Walks `[data-i18n-html]` → sets `innerHTML` from `window.I18N[code][key]`.  
Walks `[data-i18n-placeholder]` → sets `placeholder` attribute from `window.I18N[code][key]`.  
Saves `code` to `localStorage`. Updates `state.lang`.

**`computeIf('langOpen')`**  
Returns `state.langOpen`. Used to show/hide the dropdown.

**Click-outside handler**  
Added to the global `document` click listener: if `state.langOpen` and click target is outside `[data-lang-selector]`, close the dropdown.

### `render()` additions

- Keeps the language selector button label in sync with current `state.lang` (flag + code).
- Updates active/inactive styles on desktop dropdown rows and mobile lang pills.

## `i18n.js` Structure

```js
window.I18N = {
  en: {
    'nav.features': 'Features',
    'nav.pricing': 'Pricing',
    // … all keys
  },
  es: {
    'nav.features': 'Características',
    'nav.pricing': 'Precios',
    // …
  },
  de: { … },
  fr: { … },
  pt: { … },
};
```

Keys are grouped by page/section using dot notation: `nav.*`, `home.*`, `features.*`, `pricing.*`, `cases.*`, `about.*`, `contact.*`, `footer.*`, `wa.*` (WhatsApp widget), `form.*` (contact form), `errors.*` (validation messages).

## Testing

`_smoke-test.js` additions:
- Calling `setLanguage('es')` applies Spanish strings to `[data-i18n]` elements.
- `localStorage` is written with the correct code on switch.
- Switching back to `'en'` restores the original English strings.
- The `langOpen` sc-if toggles the dropdown correctly.
